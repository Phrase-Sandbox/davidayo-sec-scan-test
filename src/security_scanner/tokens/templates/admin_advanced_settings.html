Confidence thresholds

High-confidence findings are nearly certain security issues. Medium adds findings where the scanner is fairly sure but not certain. Only raise this threshold if you are seeing too many false alarms.

Warning findings are shown in the scan report but do not stop the deployment. They are useful for tracking issues you intend to fix later.

Speed tuning

A higher number makes scans faster on large repos but uses more memory. 2 is the safe default. Maximum is 16.

Which security tools should run?

Each tool scans for different types of problems. Disable a tool if it is producing too many false positives for your codebase. Disabling all tools falls back to AI-only analysis.

Which Semgrep rule packs should run?

Deselect a pack only if its rules are causing too many false positives for your project.

Report retention

When set, the scanner deletes entire scan_records and ci_scan_records rows (including any stored HTML report) that are older than this threshold. The purge runs at startup and once every 24 hours. Leave blank to keep all records indefinitely. You can also manually delete or purge records from the Reports page at any time.
